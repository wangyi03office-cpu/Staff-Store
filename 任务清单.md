# Staff Store · 代码任务清单

> 版本：v1.2（已确认）· 2026-06-21
> 依据：宪法 v1.21 + Steve v0.4 / Elon v0.5 / Rubric框架 v0.6 / 继任规格 v0.3
> 规则：按 task 逐个实现；每个 task 产出可验证的代码 + 最小测试。

## 全局决定（已确认 ✅）
- **技术栈**：Node.js + TypeScript（单仓）
- **存储**：先内存 / JSON 文件 → 跑通后上 SQLite
- **LLM / BYOK**：Anthropic Claude SDK；**初期 mock**，使全链离线可端到端跑，再接真 key
- **建造顺序**：先「运行时 + 账本 + 手种一个员工 + 店面」证闭环 → 再「Steve/Elon 工厂」证自动造
- **依赖桥接**：M2 的质量分先用 **stub**，M3 起换成真实 Rubric 评分；**排名(T6.3)待多个员工出现才有意义**，故不在 M2 做
- **运行时拓扑**：MVP = **单 Node 进程 + 一个 HTTP 服务**（店面/雇佣 API）；Steve / Elon / 闸门等以**进程内任务触发**（非分布式、非常驻 daemon）
